Сегодня я бы хотел поговорить про систему типов в языке Python.

Python — один из самых популярных языков программирования. Его популярность во многом связана с его простотой в использовании и высокой скоростью разработки. Но что значит простота использования? Многие также относят к его преимуществом "лаконичность"

Python является динамически типизированным языком и позволяет изменять типы переменных во время выполнения программы.
Например:

```python
x = 1
x = "2"
print(x) # 2
```

Существует другой тип языков, которые являются статически типизированными. Это значит, что мы не можем изменять типы переменых во время выполнения программы. Например, код на `C++` вызывает ошибку компиляции, так как мы не можем изменять тип переменной `x`:

```cpp
int main() {
    int x = 1;
    x = "2"; // error: cannot convert from 'const char *' to 'int'
    return 0;
}
```

Python также имеет сильную типизацию, это значит, что мы не можем выполнять операции с переменными разных типов. 

Например следующий код вызовет ошибку типов:

```python
y = 1
x = "2"
print(x + y) # TypeError: unsupported operand type(s) for +: 'int' and 'str'
```

Ошибочно считать, что все языки с динамической типизацией имеют слабую типизацию. Python является языком с неявной типизацией, это значит что мы можем не задавать типы переменных, во время их объявления.
Например следующий код будет работать без ошибок:

```python
x = 1
print(x)
```

При этом в языках с явной типизацией типы переменных задаются при объявлении. \
Например на `C`:

```cpp
# include <stdio.h>

int main (void) {
    char s[] = "Test String";
    float x = 0.0;
    int y = 0;

    printf ("Hello World\n");
    return 0;
}
```

Мы также можем явно задавать типы переменных, во время их объявления, для этого в Python предусмотрен специальный синтаксис аннотаций типов.

```python
<variable_name>: <type>
```

Например:

```python
x: int = 1
print(x)
```

Это отличный подход, который собственно и нравится разработчикам, которые хотели бы писать код, который был максимально читаемым и понятным. Python пытается балансировать между типобезопасностью и лаконичностью, за счёт скорости написания программ и использования статического анализа, вроде `mypy`. Аннотация типов не влияет на скорость выполнения программы, но при этом помогает находить ошибки в коде до начала выполнения, и позволяет разработчикам сделать код более читаемым и понятным. Разработчик сам решает, нужно ли использовать аннотации типов.
